Springboot只是为了省略令人头疼的模式化配置，大大缩减了开发时间提高效率，至于如何的提高，入手后和原ssm相比便会知道

搭建过程

创建Springboot项目，IDEA号称开发Java最好的工具也不是没有道理的。

  1. Create New Project / 进入http://start.spring.io创建自己项目的详情信息。



mavenProject，配置自己本地的JDK。
在选择Dependencies的时候一定要勾选web，否则启动不了。

  1. 创建成功后在根目录下一次创建如下几个包，体现是MVC架构

    controller、dao，service，entity以及经常用到的工具包utils。

    -controller，顾名思义，负责控制接收与用户输入的并调用service层的业务逻辑，（拦截）获取数据、传递数据给视图层（浏览器页面）用于视觉呈现。一般用@Controller或者@RestController来表示该类为控制层。当然这两类也是有一定的区别。比如你只想做一个简单测试，返回一串 JSON 数据，但是你用的是@Controller，这时候你可能会报错——关于视图层的错误（具体异常也是记得不太清了(っ °Д °;)っ ），解决办法为：将@Controller注释替换为@RestController。我们分析springboot源码，可以发现这两个注释间有一点区别就是，后者比前者多一个@ResponseBody（你当然可以添加@ResponseBody，但是出于懒以及要后面大量使用@ResponseBody,所以我替换了。。）。如此一来就可以不经过视图层解析而单单返回数据（JSON、xml…）。

    -dao层，最容易理解了，数据访问层，负责数据库操作。service调用dao，service就new 一个dao对象调用。事实上这儿我有一个问题，MVC设计模式为现在主流编程方式（当然也包括前端），实际上MVC本身不属于设计模式，它讲述的是一种结构，最终的目的是解耦，最终流程走向为：表示层调用控制层，控制层调用业务层，业务层调用数据访问层。对，目的是为了解耦，但是如果直接在业务层new一个dao对象来访问数据库，这不就存在了具体的对象而非引用造成耦合的情况吗，万一这个对象不存在了，我还是需要修改业务逻辑。所以并未完全实现解耦。关于这点我还是要继续学习。

    -service层，负责处理具体的业务逻辑、操作以及与数据层的交互。

    -entity层，实体嘛，当然是实体类啊，字段与数据库最好是对应的，不然不太熟的话，很容易弄混，另外数据库字段的类型，与实体类也有讲究，比如varchar = string, smallint = int/integer…需要注意的是sql.timestamp这类类型变量，也许你的数据库中有两个需要设置时间的地方，比如注册时间以及最后登录时间，注册时间不用说是currentTime，但是最后注册时间不能，同张表只能有一个变量是currentTime，另一个只能设置为NULL，当然这里最后编译的时候还可能会出现一个错误，比如00：00：00-000….（(￣▽￣)”老年人记性不要在意），这里你需要在你的数据库配置url那儿尾部加上&zeroDateTimeBehavior=convertToNull即可完美解决。

    -utils工具包，内含后面用到的分页啊，md5摘要加密啊等等，因为是通用的，所以最后还是将其分离出供其他类引用，贯彻MVC结构。
  2. 最后需要配置你的数据库。等下，你不是说Springboot免去一大堆配置吗，为什么还要配置？！的确，但是必要的配置你还是要去配的，数据库就是其中之一。贴上自己的代码：
    #MysqlConfig
spring.datasource.url=jdbc:mysql://localhost:3306/ori?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull
spring.datasource.username=root  //账号密码字段自定义(注释方式非://而是#)
spring.datasource.password=123456    //习惯加便于说明
spring.datasource.driver-class-name=com.mysql.jdbc.Driver
#MybatisCinfig
mybatis.type-aliases-package=top.zexus.ori.entity  //免去重复路径
mybatis.mapper-locations=classpath:mapper/*.xml  //扫包
#RedisConfig

  3. 其实吧，Springboot基础项目我们早已创建好了，只不过建以上的几个包，原因也说得很明白。就算你包里什么东西也没有，也照样跑的通，只不过在浏览器中输入localhost:8080是一片空白的。不如我们来跑一遍吧。

    一般来说，一个Springboot项目创建时便内置了它的启动器，有的情况下则没有。这时我们来手动创一个，左上角工具栏（what？第一次用IDEA没有工具栏咋办！(＃°Д°) 简单，就像把大象放进冰箱一样只需三步，而我们只需要两步即可呼出工具栏：菜单栏View下拉–>勾选toolbar，其他栏视自身需要选勾，本人秉承一劳永逸原则全给勾了，很是睿智o(￣▽￣)o）绿色倒三角（run键）左边有一块空的地方，下拉选择Edit Configurations...左上角蓝色+号（这里由于我喜欢material design风格，所以下载了插件material theme UI，选择的Oceanic样式，个别图标风格不一），往下拖，选择Spring Boot的绿色叶子样式的选项，最上方给它取个名儿，main class那儿点击...自动扫描整个项目（实际上Springboot项目的启动类上会有一个@SpringBootApplication这个注解，意思是表明这个类为全项目启动入口，扫的是这个注解）。配置完成后，启动，可以看到控制台上出现Springboot样式的类似于颜文字的标识，所以成功惹！

    当然你在浏览器是看不到的，因为我们这里面啥也没有！那你让我看一篇空白吗？？？当然不，所以我们现在写一个测试类，亲身体会MVC的思想。

当然，也有的小伙伴不想用内置的，考虑到以后要分布式外接tomcat，行，没问题，配一个即可~不过本项目为单机，分布式以后还要重构，先从入门说起。一开始我是头铁不想用内置tomcat，自己重新搞一个，结果跑的时候会报错，这是我就奇怪了，正常启动但是页面不显示什么鬼？？我看了一下日志：某个tomcat的jar包未被加载。我在怀疑是不是版本号问题，因为我用的是本地tomcat7.0版本，而内置tomcat版本号为8.5版本，导致找不到使用默认版本，我还能说啥？：）

新建工具类Result于utils包下，为了把数据进行进一步包装，按照我们想要的格式返还。

public class Result implements Serializable {
    private int status;
    private String msg;
    private Object data;

public int getStatus() {
    return status;
}

public void setStatus(int status) {
    this.status = status;
}

public String getMsg() {
    return msg;
}

public void setMsg(String msg) {
    this.msg = msg;
}

public Object getData() {
    return data;
}

public void setData(Object data) {
    this.data = data;
}

@Override
public String toString() {
    return "Result{" +
            "status=" + status +
            ", msg='" + msg + '\'' +
            ", data=" + data +
            '}';
}

}
