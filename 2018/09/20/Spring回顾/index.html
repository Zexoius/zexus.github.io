什么是Spring

  • Spring 是一个轻量级Java开源框架，适用于JavaSE/EE，也就是一站式 FullStack 。
  • 使用 JavaBean 完成以前只可能用 EJB 完成的事。
  • 核心：控制反转（IOC）和面向切面（AOP）。

    EE 开发分成三层结构：
    表现层（UI）、业务逻辑层（BLL）、数据访问层（DAL），目的为了实现“高内聚，低耦合”的思想。
    后面提到 MVC 的时候详细解释，本章重点解释什么是 IOC 与 AOP 。

什么是 IOC ？

IOC:Inversion of control，控制反转。它并不是一种技术，而是一种设计思想。使用 IOC 意味着将自己设计好的对象（类）交给容器控制，而非传统的在对象内部直接控制，为的是“高内聚，低耦合”的软件设计思想。

如何理解 IOC ？谁控制谁？控制什么？怎么控制？为什么要反转控制？

顺带讲述一个人对我学习思想的重要塑性方法：凡事从三个方面考虑：这是什么，怎么做，为什么。

  • 谁控制谁？

    传统 JavaSE 程序设计内，程序猿直接在对象内部 new 构造对象（单身汪的悲哀），是程序主动创建依赖对象。而 IOC 通过容器来控制对象的创建。即：IOC 控制对象，控制外部资源的获取（不仅仅是对象，更是包括了文件）。

  • 为什么要反转？

    先说说正转吧，传统应用程序是由我们自己在对象中主动控制去直接获取依赖对象。而反转，则是由容器来帮忙创建及注入依赖对象。因为是由容器帮我们查找及注入依赖对象，而对象只是被动的接受以来对象，是为反转。在哪儿反转了呢？以来对西昂的获取被反转了。

概念太模糊？

ok，我们举几个栗子吧。

假设！我们要设计两个类：Girl、Boy。Boy.class 内有一个kiss()。好了，问题来了，我们的 Boy 应该如何认识这个 Girl ?

在我国，常见的GG与MM认识的方法有一下几种：

  1. 青梅竹马
  2. 亲友介绍
  3. 父母包办

那么，哪一种才是最好的呢？

青梅竹马：Boy 打小就很了解这个 Girl 。

我们创建 Girl 的缺点暴露出来了：无法更换！！而且要负责 Girl 的整个生命周期。可是，我想换一个肿么办？（虽然说不支持常换，但是不适合就算了吧。。）

亲友介绍：找个媒婆负责找 Girl 来见。

没有错，我不满意，我换一个就好了，天涯何处无芳草呢？但是呢，这个亲友 GirlFactory 经常以 Singleton 的形式出现，不然就是Globals，无处不在，无所不能。繁琐、不够灵活。我为什么一定要一个亲友掺和进来呢？为什么一定要给他介绍费呢？万一亲友爱上了我的女票呢？？？

父母包办：不费吹灰之力，躺着等着被 kiss 就好了。

ok，这就是对于 Boy 来说是最好的方法，只要想办法，贿赂了 Boy 的方法，并把Girl 交给他，那么，Boy 就可以轻轻松松和 Girl kiss 了。说实话，中华传统强无极，至少不用双方瞎忙乎了。

父母包办 = IOC，将对象的创建和获取提取到外部，由外部容器提供需要的组件。

什么是AOP

AOP（面向切面编程 Aspect Oriented Programming）是OOP（面向对象编程 Object Oriented Programming）的补充与完善。由于OOP中存在大量与业务逻辑无关却为业务逻辑所共同调用的逻辑（切面），所以为了减少重复性，降低耦合度，AOP将这些重复的封装起来，提高可操作性、可维护性。

核心概念：横切关注点（对哪些方法拦截，之后如何处理）、切面（对横切关注点的抽象）、连接点（被拦截到的方法，也可以是字段，构造器）、切入点（对连接点进行拦截的定义）、通知（拦截到连接点之后执行的代码块，分为：前置、后置、异常、最终、环绕）、目标对象（代理的目标对象）等概念。
